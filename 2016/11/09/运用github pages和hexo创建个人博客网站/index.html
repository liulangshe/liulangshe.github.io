###1.什么是github pages

####先了解什么是 GitHub？

确切的说 GitHub 是一家公司，位于旧金山，由 Chris Wanstrath, PJ Hyett 与 Tom Preston-Werner 三位开发者在2008年4月创办。地址：How people build software · GitHub ，主要提供基于git的版本托管服务。一经上线，它的发展速度惊为天人，截止目前，GitHub 已经发展成全球最大的开（同）源（性）社区。

GitHub Pages本用于介绍托管在GitHub的项目， 不过，由于他的空间免费稳定，用来做搭建一个博客再好不过了。

github Pages可以被认为是用户编写的、托管在github上的静态网页。

####为什么选择GitHub Pages？

很多人用wordpress，为什么要用github pages来搭建？

  • github pages有300M免费空间，资料自己管理，保存可靠；
  • 学着用github，享受github的便利，上面有很多大牛，眼界会开阔很多；
  • 顺便看看github工作原理，最好的团队协作流程；
  • 帮助文档详细而友好；
  • github是趋势；
  • 吉祥物太萌-0-（喵）

###2.什么是hexo

快速、简单且功能强大的 Node.js 博客框架。A fast, simple & powerful blog framework, powered by Node.js.–Hexo作者 @tommy351

Hexo 是一个简单地、轻量地、基于Node的一个静态博客框架，可以方便的生成静态网页托管在github.

###3.环境部署

  1. Node 点击进入官网下载，一路确认即可。
  2. Git 点击进入下载页面，根据自己的系统下载，安装过程中出现选项一般选择第一个或者默认。

###4.安装hexo
可以事先参阅 官方文档
如果你的Git安装成功的话鼠标右键桌面会出现git bash here，打开。
在某个盘（D盘为例）内新建hexo文件夹

cd d:/hexo

下载安装hexo

npm install -g hexo-cli

初始化hexo

hexo init blog

并且：

cd blog
npm install

以上基本完成了hexo的安装。
这里有必要提下Hexo常用的几个命令：

  • hexo new “postName” #新建文章
  • hexo new page “pageName” #新建页面
  • hexo generate #生成静态页面至public目录
  • hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
  • hexo deploy #将.deploy目录部署到GitHub
  • hexo deploy #将.deploy目录部署到GitHub
  • hexo help # 查看帮助
  • hexo version #查看Hexo的版本

命令的简写为：

hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy

###5.注册github

  • 打开官网 github
  • 选择Sign Up注册账号
  • [建议：名字取成熟的，邮箱不要qq的，密码用不会忘记的]
  • step2选择个人免费账号，下面这个不用勾选，然后一路完成吧！
  • 其他的就不再重复造轮子了，想了解github更多内容可以参阅stormzhang的博客。
    ####噢，别忘了New repository，项目名为：你的用户名.github.io
    注意：必须为这个名字，这以后就是访问你博客的域名了，就像本站的域名：https://liulangshe.github.io/

###6.两者关联

####配置文件
在D盘hexo文件夹中有_config.yml文件，用Notepad++打开（憋说你没有：Notepad++）
在末尾按照格式添加

deploy:
type: git
repo: https://github.com/你的用户名/你的用户名.github.io.git 
branch: master

#本来就有deploy，你要做的就是按照你的用户名填写补充完整保存即可。

####配置SSH

  • 打开Git Bash终端：在桌面右键，会出现”Git Bash here”的选项，点击即可。
  • 设置user name和email:
git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub注册邮箱"
  • 生成ssh密钥：
    ssh-keygen -t rsa -C "你的GitHub注册邮箱"
一般情况下是不需要密码的，所以，接下来直接回车就好。
  • 复制公钥：
    建议下载一个实用软件：Everything
    使用该软件直接搜索id_rsa.pub
    注：id_rsa文件是私钥，要妥善保管，id_rsa.pub是公钥文件。
    同样的，用Notepad++打开。
  • 添加公钥到github：

    在Settings内点击SSH and GPG keys选项，然后点击New SSH key(新建SSH)按钮；
    将id_rsa.pub中的内容复制到Key文本框中，然后点击Add SSH key(添加SSH)按钮；
    Title随意。
  • 测试SSH：
ssh -T git@github.com

若返回：

The authenticity of host 'github.com (207.97.227.239)' can't be established. 
RSA key fingerprint is 17:24:ac:a5:76:28:24:36:62:1b:36:4d:eb:df:a6:45.
Are you sure you want to continue connecting (yes/no)?

输入yes后回车。
然后显示如下信息则OK(其中的liulangshe是用户名)。

Hi liulangshe! You've successfully authenticated, 
but GitHub does not provide shell access.

基本完成以上几个步骤 在hexo文件夹下Git bash

hexo clean
hexo generate
hexo deploy

即可访问https://你的用户名.github.io/
